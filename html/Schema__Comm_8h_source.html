20// Ces objets stockent un graphe de conversation entre
21// processeurs : chaque proc. possede une liste de processeurs a qui
22// envoyer et une liste de processeurs de qui recevoir.
23//
24// Le groupe est l'ensemble des processeurs sur lesquels on GARANTIT
25// que les methodes suivantes seront appelees SIMULTANEMENT sur tous
26// les processeurs du groupe :
27// - begin_comm()
28// - set_send_recv_pe_list(...)
29// - echange_taille_et_messages()
30// - end_comm()
31//
32// En particulier : Il est interdit d'utiliser ces methodes a
33// l'interieur d'une boucle qui n'est pas executee le meme nombre de
34// fois par tous les processeurs du groupe, ni a l'interieur d'un
35// "if() { }" dont le deroulement n'est pas identique sur tous les
36// processeurs du groupe.
37//
38// Il est interdit de commencer une nouvelle communication pendant
39// qu'une autre est en cours (une communication finit toujours par
40// "end_comm()")
41// Interdiction en particulier d'utiliser "envoyer", "recevoir" et les
42// fichiers disques partages entre le premier appel a schema.send_buffer()
43// et "terminer()". Faire tres attention a toutes les methodes qu'on utilise
44// entre ces deux appels !
45//
46// Sur un PE donne, une sequence d'echange doit etre construite comme suit:
47// schema.begin_comm()
48// schema.send_buffer(pe1) << data_to_send;
49// schema.send_buffer(pe2) << data_to_send;
50// ...
51// schema.echanger_taille_et_messages();
52// schema.recv_buffer(pe2) >> data_to_recv;
53// schema.recv_buffer(pe3) >> data_to_recv;
54// ...
55// schema.end_comm();
56//
57// La communication n'est pas forcement symetrique : un processeur peut envoyer
58// un message a un processeur et recevoir d'un autre.
59// En revanche, l'utilisateur garantit que les listes de processeurs fournies dans
60// send_pe_list et recv_pe_list verifient le principe "tu m'ecoutes quand je te parle !"
61// (c'est a dire qu'un processeur A appartient a send_pe_list sur le processeur B
62// si et seulement si le processeur B appartient a recv_pe_list sur le processeur A).
63
64// Modif BM 20/06/2013: j'ajoute set_all_to_allv_flag. Si le drapeau est mis, le schema de
65// communication utilise MPI_alltoallv au lieu de ISend IRecv. Pour tenter de resoudre
66// les problemes rencontres sur supermuc dans la routine de lecture ecriture fichiers
67// (schema tout le monde ecrit au processeur 0 => erreur d'allocation des MPI_requests).
