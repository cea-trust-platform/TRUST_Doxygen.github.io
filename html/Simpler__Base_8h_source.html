34// Description
35//
36// Classe de base des classes dediees a la resolution implicite d une equation
37// L equation est discretisee sous la forme suivante :
38// (M/dt + C(U) + D)Xn+1 = Sv + Ss + (M/dt)Xn
39// avec
40// C(U) matrice de convection
41// D matrice de diffusion
42// M matrice de masse
43// Sv terme source volumique
44// Ss terme source surfacique (condition de Neumann)
45// Xn et Xn+1 designe l inconnue respectivement aux temps tn et tn+1
46//
47// A chaque etape en temps (tn) on va traiter la resolution du systeme suivant :
48// A[Xk] = b[Xk]
49// avec
50// A[Xk] = (M/dt + C(Uk) + D)
51// b[Xk] = Sv + Ss + (M/dt)Xk
52// k designe un indice d iteration pour la resolution iterative du systeme matriciel (tn fixe)
53
54// La methode de resolution employee pour determiner X est une methode de point fixe :
55// On pose f(Xk) = A[Xk]Xk -b[Xk] ainsi Xsol solution verifie f(Xsol) = 0
56// Le developpement limite au premier ordre donne la relation :
57// Xk - Xk+1 = f(Xk)/f'(Xk)
58// En faisant l hyposthese que f' = A (pas vrai en toute rigueur pour la partie convection)
59// le systeme s ecrit : A[Xk](Xk-Xk+1) = f(Xk) ou encore
60// A[Xk]Xk+1 = A[Xk]Xk - (A[Xk]Xk-Ss) + Sv +(M/dt)Xk
61//
62//
63// Methode iterer_eqn() pour traiter une equation autre que Navier_Stokes
64// - fait construire la matrice (matrice) et le second membre (resu) par l equation (assembler_avec_inertie(...))
65// - declenche la resolution du systeme matriciel (le_solveur_.resoudre_systeme(...))
66
67// Methode iterer_NS() appelee par iterer_eqn() pour traiter le cas specifique de l equation de Navier_Stokes
68// Les algorithmes disponibles sont : Simple - Simpler - Piso - Implicite
69// Voir classes filles de Simpler_base pour la description des algorithmes respectifs
70
