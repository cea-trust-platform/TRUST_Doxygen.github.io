38//! Classe de base des formats de postraitements pour les champs<br>(lata, med, lml, meshtv).<br>Utilisation de la classe par l'interface generique:<br>-typer un objet format de postraitement<br>-initialiser l objet format de postraitement<br>-proceder a des operations de complement et verification<br>-ecrire un domaine<br>-ecrire_temps<br>-ecrire_champ (un type de champ supporte: aux elements ou aux sommets,<br>ou encore aux faces si on a ecrit les faces du domaine<br>et si les faces sont supportees par le postraitement)
39
40// Les parametres intervenant (eventuellement) dans une operation de postraitement
41//
42// Nom nom_fichier : nom du fichier sur lequel on ecrit
43// int champ : 1 si postraitement de champ, 0 sinon
44// int stat : 1 si postraitement de statistiques, 0 sinon
45// double dt_ch : periode de postraitement des champs
46// double dt_stat : periode de postraitement des statistiques
47// int reprise : 1 si reprise effectuee, 0 sinon
48// int axi : 1 si calcul axi, 0 sinon
49// int est_le_premier_post : 1 si premier postraitement pour un fichier d ecriture donne, 0 sinon
50// int est_le_dernier_post : 1 si dernier postraitement pour un fichier d ecriture donne, 0 sinon
51// Domaine dom : domaine de calcul
52// Nom id_domaine : nom du domaine
53// Nom id_zone : nom de la zone
54// IntVect faces_som : connectivites faces-sommets
55// IntVect elem_faces : connectivites elements-faces
56// int nb_som : nombre de sommets du domaine dom
57// int nb_faces : nombre de faces de la zone
58
59// double t_init : temps initial du calcul
60// double temps_courant : temps courant du calcul
61// double temps_champ : temps du champ cible
62// double temps_post : temps de l objet postraitement (ecriture si temps_post<temps_courant)
63
64// Nom id_champ_post : identifiant du champ generique (nom ou composante)
65// Nature_du_champ nature_champ : scalaire, multi_scalaire, vectoriel
66// int nb_compo : nombre de composantes du champ
67// Noms nom_compos : vecteur contenant le nom des composantes du champ
68// Noms unites : vecteur contenant les unites du champ a ecrire
69// Motcle loc_post : localisation : ELEM, SOM, FACES
70// int ncomp : numero de composante (-1 si ce n est pas une composante)
71// DoubleTab data : tableau de valeurs a ecrire sur fichier
72//
73
74// Rq: data
75// Si on postraite le champ et non pas une composante d un champ
76// Si le tableau de valeurs du champ est a une seule entree, data doit etre dimensionne en data(nb_ddl,1)
77// Si on postraite une composante d un champ
78// Si le tableau de valeurs fourni est a plusieurs entrees, data doit etre dimensionne en data(nb_ddl)
79
